# HTML/PDF company-specific insights deliverables
- Chip/pill/tag vertical centering: every chip-style component (tier tags, field labels, gap-type pills, use-case tags, cover pills) must vertically center its label text within the pill. Use display:inline-flex with align-items:center and line-height:1 (plus vertical-align:middle), not inline padding alone, which can leave the text visually off-center.
- Inline-citation findability: every inline citation must let the reader reach the specific cited evidence when clicked. When citing congress abstracts that share one supplement DOI, do not link each abstract identifier to a generic supplement landing page or a section-level DOI where the specific abstract cannot be located; instead link each abstract identifier to a target where that exact abstract is findable, for example the open-access abstract book PDF opened at the abstract's page via a #page=N fragment (per-abstract page numbers are available in the source data).
- Inline-citation re-verification: before finalizing, re-verify that every inline citation link actually resolves and that the linked page contains the cited claim, figure, or abstract number; correct any link that does not, or mark it [citation needed].
- Abstract detail modals: use official source metadata labels rather than generic repeated labels such as "Record"; do not expose internal extraction-status labels such as "structured record" unless they are official source text; render author and affiliation numeric markers as superscripts, put each numbered affiliation on its own line, and style section headings such as Background, Methods, and Results so they are visually distinct.
- Abstract ID citation UX: in generated conference or abstract pages, do not show visible page-number text immediately next to hyperlinked abstract IDs when the link or popup already resolves to the abstract. When local abstract detail data or a modal exists, make the abstract ID open the in-page detail popup as the primary action; use PDF page links only as secondary or contextual evidence links.
- Author/KOL profile extraction: when generating profile data from PDF abstract books, discard profile labels contaminated by affiliation or institution text, and spot-check high-count profiles against raw author strings before presenting them as author activity.
- KOL question wording: write questions that invite discussion, process-sharing, and rationale rather than sounding pushy or unrealistic; avoid asking for an "exact" threshold or similarly over-precise answer when expert judgment is likely contextual.
- Card-grid alignment: top-align card content across columns and rows, especially when cards have different title lengths or body copy.
- Multi-column table fit: for generated conference/abstract pages, adjust column widths to fit tables within the main content area on desktop before using a horizontal scroller; reserve horizontal scrolling for genuinely narrow mobile/tablet viewports.
- Chip/pill centering with padding: when chips, pills, tags, or badges carry vertical padding, center the label text with display:inline-flex and align-items:center plus a normalized line-height and symmetric top/bottom padding; do not rely on inline-block with padding alone, which can leave text visually off-center, especially with geometric fonts.
- Inline-citation deep-linking and parallel re-verification: link each citation to a target that resolves to the specific record, not a generic publication or supplement DOI where the abstract or poster number is not findable; for example, deep-link to the searchable abstract archive with a ?q=<ABSTRACT-NUMBER> query parameter. When the user reports citation problems, re-verify every inline citation with fresh subagents working in parallel, grouped by source type (journal/PubMed, ClinicalTrials.gov, regulator/company pages), confirming each link resolves and that the landing page supports the specific cited figure.
